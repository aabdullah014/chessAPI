Welcome to taskAPI!

Structure of the API:

Families

  • Families are the most general group, and each family has tasks.
  • Each family has a specific id it is associated with. That id is used when delegating tasks to keep track of which tasks belong to which families.

Tasks

  • Tasks are pretty straightforward in their function, but there are things to keep in mind.
  • You must be registered when creating a new task.
  • Each task requires a name, a due date, and a family id to associate it with a family. When a task is created, it will automatically show up in the associated family's task list.

Endpoints

    Registering/Members:

    • POST /register
    • POST /auth

    Tasks:

    • GET /tasks
    • GET /task/(taskname)
    • POST /task/(taskname)
    • PUT /task/(taskname)
    • DELETE /task/(taskname)

    Families:

    • GET /families
    • GET /family/(familyname)
    • POST /family/(familyname)
    • DELETE /family/(familyname)
